# Upwork Voice AI Job Bot

A backend service that automatically scrapes Upwork for Voice AI jobs and delivers them to your Telegram. When a job lands in your chat, tap a button to generate a personalised cover letter on demand using Groq AI.

---

## How it works

```
Upwork Jobs API → keyword filter → Neon Postgres (dedup) → Telegram card + button
                                                                        ↓ (on button press)
                                                               Groq → draft proposal
```

1. A **cron job** runs every N minutes (default: 30)
2. It searches Upwork for your configured queries via the **official Upwork Jobs API v2**
3. Results are filtered locally by a **Voice AI keyword list** — no AI cost, zero latency
4. New jobs (not yet seen in the database) are saved to **Neon Postgres** and sent to **Telegram**
5. Each Telegram card has a **📝 Draft Proposal** button — pressing it makes a single **Groq** call and edits the message to show the cover letter

| Layer | Technology |

## Prerequisites

- Node.js 22+
- A [Neon](https://neon.tech) account (free tier works)
- A [Groq](https://console.groq.com) account (free tier works)
- A Telegram bot token from [@BotFather](https://t.me/botfather)
- An [Upwork Developer](https://developers.upwork.com) account with an approved app

## Setup

### 1. Clone and install

```bash
git clone https://github.com/Demilade01/upwork-scrapper.git
cd upwork-scrapper
npm install
```

### 2. Configure environment variables

```bash
cp .env.example .env
```

Fill in `.env`:

```env
# Telegram
TELEGRAM_BOT_TOKEN=       # From @BotFather
TELEGRAM_CHAT_ID=         # Your chat or group ID (see below)

# Groq
GROQ_API_KEY=             # From console.groq.com → API Keys

# Neon Postgres
DATABASE_URL=             # From neon.tech → Connection Details → Node.js string

# Upwork API
UPWORK_CLIENT_ID=         # From developers.upwork.com → your app
UPWORK_CLIENT_SECRET=     # From developers.upwork.com → your app
UPWORK_REFRESH_TOKEN=     # Generated by: npm run setup-oauth

# Scheduler
POLL_INTERVAL_MINUTES=30

# Your profile (used in proposal drafts)
FREELANCER_NAME=
FREELANCER_SKILLS=
FREELANCER_EXPERIENCE_YEARS=
FREELANCER_BIO=

# Search queries (comma-separated)
UPWORK_SEARCH_QUERIES=voice ai agent,voice ai architect,conversational ai voice,...
```

---

### 3. Get your Telegram Chat ID

1. Start your bot (send it any message)
2. Open in browser (replace with your token):
   ```
   https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
   ```
3. Find `"chat":{"id": 123456789}` — that number is your `TELEGRAM_CHAT_ID`

> For a **group or channel**: add the bot to the group, send a message, then check `getUpdates`. The ID will be negative (e.g. `-1001234567890`).

---

### 4. Register your Upwork app

1. Go to [developers.upwork.com](https://developers.upwork.com)
2. Create a new application
3. Set the **Callback URL** to: `http://localhost:3000/callback`
4. Copy the **Client ID** and **Client Secret** into `.env`

---

### 5. Get your Upwork refresh token (one-time)

```bash
npm run setup-oauth
```

This starts a local server on port 3000 and prints an authorisation URL. Open it in your browser, approve access, and the script prints your `UPWORK_REFRESH_TOKEN`. Paste it into `.env`.

> The refresh token is long-lived. You only need to do this once (or if you revoke app access).

---

### 6. Run locally

```bash
npm run dev
```

Expected output:

## Usage

**Job alerts** land in Telegram automatically every poll interval.

Each card shows:
- Job title
- Budget / hourly rate
- Required skills
- Client country
- Posted time
- Description preview (600 chars)
- Direct link to apply on Upwork
- **📝 Draft Proposal** button

**Pressing the button:**
1. Groq generates a tailored cover letter (< 180 words) using your profile
2. The message is edited in-place to show the proposal
3. The button is removed

> Groq is **only called when you press the button** — no quota is consumed on jobs you ignore.

## Deployment (Heroku)

```bash
# One-time setup
heroku create your-app-name
heroku config:set \
  TELEGRAM_BOT_TOKEN=... \
  TELEGRAM_CHAT_ID=... \
  GROQ_API_KEY=... \
  DATABASE_URL=... \
  UPWORK_CLIENT_ID=... \
  UPWORK_CLIENT_SECRET=... \
  UPWORK_REFRESH_TOKEN=... \
  POLL_INTERVAL_MINUTES=30 \
  FREELANCER_NAME="Your Name" \
  FREELANCER_SKILLS="Voice AI, LiveKit, Deepgram, ElevenLabs" \
  FREELANCER_EXPERIENCE_YEARS=5 \
  FREELANCER_BIO="Your bio here" \
  UPWORK_SEARCH_QUERIES="voice ai agent,voice ai architect,livekit voice"

# Deploy
git push heroku main
heroku ps:scale worker=1

# Monitor
heroku logs --tail
```

> The `Procfile` runs the compiled output: `worker: node dist/main.js`. Heroku automatically runs `npm run build` (via the `heroku-postbuild` script or buildpack) before starting the dyno.

Add this to `package.json` scripts for Heroku auto-build:
```json
"heroku-postbuild": "npm run build"
```

## Adding more search queries

Edit `UPWORK_SEARCH_QUERIES` in `.env` (or Heroku config vars) — comma-separated:

```env
UPWORK_SEARCH_QUERIES=voice ai agent,voice ai architect,conversational ai voice,realtime voice agent,ai phone agent,voice bot twilio,deepgram elevenlabs,livekit voice,vapi retell ai,ai calling agent
```

The scraper runs all queries **concurrently** and deduplicates results by job ID within each run.

## Voice AI keyword filter

Before any job is stored or sent to Telegram, it must match at least one keyword from a built-in list (in `scraper.ts`). This prevents unrelated jobs from slipping through even when queries are broad. Current keywords include:

`voice ai` · `voice agent` · `voice bot` · `conversational ai` · `deepgram` · `elevenlabs` · `livekit` · `vapi` · `retell ai` · `openai realtime` · `twilio voice` · `webrtc audio` · `sip trunk` · `ivr` · `telephony` · `tts` · `speech-to-text` · and more
